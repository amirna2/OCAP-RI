This package contains components comprising the presentation layer of the JMF implementation. The presentation layer is used by the players to manage the presenation of content. The elements in this package can be divided into the following categories:
  • Presentation Interfaces — define the contract that presentations must provide for players. It includes these interfaces:
    • {@link Presentation} — base functionality supported by all presentation types.
    • {@link VideoPresentation} — extends Presentation to define additional functionality needed by presenations that use the video device.
    • {@link ServicePresentation} — extends VideoPresenation to define additional functionality needed by presentations that decode an MPEG stream.
  • Framework Classes — abstract classes upon which the presentations are implemented:
    • {@link AbstractPresentation} — base class of presentation class hierarchy.
    • {@link AbstractVideoPresenation} — base class of all presentations that use the video device.
    • {@link AbstractServicePresenation} — base class of presentations that present an OCAP Service.
    • {@link AbstractDVRServicePresentation} — base class of presentations that present an OCAP Service from a time-shift buffer or recording.
  • Implementation Classes — concrete classes that implement presentations for various content types:
    • {@link AudioPresentation} — extends AbstractPresentation to provide an audio presentation.
    • {@link DripFeedPresentation} — extends VideoPresentation, providing a drip-feed video presentation.
    • {@link BroadcastServicePresenation — extends AbstractServicePresentation, providing a presentation of a broadcast OCAP Service.
    • {@link TSBServicePresenation} — extends AbstractDVRServicePresentation to handle an OCAP Service presentation through a time-shift buffer.
    • {@link RecordedServicePresentation} — extends AbstractDVRServicePresentation to handle an OCAP Service presentation of a RecordedService.
    • {@link SegmentedRecordedServicePresentation} — extends RecordedServicePresentation to handle an OCAP Service presentation of a SegmentedRecordedService.
  • Presentation Context Interfaces — define the contract that the environment in which a presentation runs must support. These interfaces are implemented by various player classes that use the presentations. Here are the context interfaces:
    • {@link PresentationContext} — base context that must be supported for all presentation types.
    • {@link AudioPresentationContext} — context that is needed by an {@link AudioPresentation}.
    • {@link VideoPresentationContext} — context that is needed by an {@link AbstractVideoPresentation}.
    • {@link DripFeedPresentationContext} — context needed by a {@link DripFeedPresentation}.
    • {@link ServicePresentationContext} — context needed by an {@link AbstractServicePresentation}.
    • {@link DVRServicePresentationContext} — context needed by an {@link AbstractDVRServicePresentation}.
