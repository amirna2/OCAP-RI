This package contains player framework classes upon which the various player implementation classes are built. For instance, this package contains {@link AbstractPlayer}, the abstract base class of the entire player implementation hierarchy. There are also base classes for different categories of players:
  • {@link AbstractVideoPlayer}
    • — for players that use the video device.
  • {@link AbstractAudioPlayer}
    • — for players that only use audio.
  • {@link AbstractServicePlayer}
    • — for players that decode an MPEG transport stream.
This package also contains several player implementation classes, some of which are subclassed by the players in the org.cablelabs.impl.media.content packages, allowing them to be instantiated via the createPlayer() method of the {@link javax.media.Manager} class:
  • {@link BroadcastServicePlayer}
    • — basic player for playing an MPEG transport stream from a network interface.
  • {@link BroadcastServiceMediaHandler
    • — version of BroadcastServicePlayer that can be used by a ServiceContext as a ServiceContentHandler.
  • {@link DripFeedPlayer}
    • — basic dripfeed player implementation.
  • {@link TSBServiceMediaHandler}
    • — player to decode MPEG transport stream through a time-shift buffer for trick-mode playback.
  • {@link RecordedServicePlayer}
    • — decodes MPEG transport stream from a RecordedService.
  • {@link RecordedServiceMediaHandler}
    • — version of RecordedServicePlayer that can be used by a ServiceContext as a ServiceContentHandler.
  • {@link SegmentedRecordedServicePlayer
    • — decodes MPEG transport stream from a SegmentedRecordedService.
  • {@link SegmentedRecordedServiceMediaHandler
    • — version of SegmentedRecordedServicePlayer that can be used by a ServiceContext as a ServiceContentHandler.
Lastly, this package contains ancillary helper classes that are used by the above classes:
  • Clock-related classes: {@link PlayerClock}, {@link AlarmClock}, {@link ClockImpl}, and {@link MediaTimeBase}.
  • Video-related classes: {@link VideoDevice}, {@link VideoTransformationDfc}, and {@link PlayerAssociationControl}.
  • Service-related classes: {@link ServiceAdapter}, {@link Context}, {@link Authorization}.
  • The general utility class {@link Util}.
