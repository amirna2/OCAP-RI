The org.cablelabs.gear.data package contains the classes that are used to manipulate various data resources in GEAR applications. While this package is largely made up of data wrapper types and implementations, it is also home to the {@link org.cablelabs.gear.data.Locator data locator} API used to locate data resources.

Package Specification

Data Wrappers

The data wrapper types all implement the {@link org.cablelabs.gear.data.DataWrapper} interface. This simple interface is simply used to "tag" a type as data wrapper. The package is composed of the following types of data wrappers:
  • {@link org.cablelabs.gear.data.GraphicData}
  • {@link org.cablelabs.gear.data.AnimationData}
  • {@link org.cablelabs.gear.data.SoundData}

Graphic Data

The GraphicData interface defines an API for the management and rendering of graphic pixel data. It provides methods for drawing and determining the size of graphic data. Each implementation of the GraphicData interface is based on a different type of wrapped data. The simplest is the {@link org.cablelabs.gear.data.ImageGraphic}, which is a data wrapper around a simple Image object. The {@link org.cablelabs.gear.data.PortfolioGraphic} is a wrapping around a {@link org.cablelabs.gear.util.PicturePortfolio} object and a single String id.

Animation Data

The AnimationData interface defines an API for the management and rendering of animation frame data. It provides methods for drawing individual frames, as well as determining the number of animation frames, and the maximum size of all frames. As with GraphicData, each implementation of the AnimationData interface is based on a different type of wrapped data. The simplest is the {@link org.cablelabs.gear.data.ImageAnimation}, which is a data wrapper around a simple Image[] object. The {@link org.cablelabs.gear.data.PortfolioAnimation} is a wrapping around a PicturePortfolio and a set of String ids. The {@link org.cablelabs.gear.data.GraphicAnimation} class is the most versatile: it is a wrapping around an array of GraphicData objects.

Sound Data

The SoundData class is an extension to the HAVi HSound class. It makes the locator used to load the sound data an intrisic part of the object.

Data Locators

The data locator types all implement the Locator interface. This simple interface is used to provide for various ways of specifying the location of a data resource. It is essentially a wrapper around a java.net.URL, but allows for dynamic construction of that URL.

The package is composed of the following Locator implementations:

  • {@link org.cablelabs.gear.data.ResourceLocator}
  • {@link org.cablelabs.gear.data.FileLocator}
  • {@link org.cablelabs.gear.data.URLLocator}
